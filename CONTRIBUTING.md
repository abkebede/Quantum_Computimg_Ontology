# Contributing to the QC Code Study Library

Thank you for contributing. This project is designed as a beginner-friendly and faculty-friendly quantum computing code and ontology library. Contributions should help learners understand working code, not merely collect files.

## What can be contributed

You may contribute:

1. Original working quantum computing code.
2. Adapted code from openly licensed sources, with source and license clearly recorded.
3. Metadata cards for existing examples.
4. Line-by-line explanations.
5. Application notes connecting code examples to mathematical physics, general physics, chemistry, optimization, materials, or computing problems.
6. Corrections to code, documentation, or ontology classifications.

## Contribution requirements

Every new code contribution should include:

- the working code file;
- a short explanation file, preferably Markdown;
- a metadata card using `ontology/qc_code_metadata_schema.yaml`;
- source and license information;
- expected output;
- safe modifications for learners;
- application connection.

## License and attribution

Do not paste code from another repository unless the license permits reuse. If the license is unclear, link to the source and write an original teaching version instead.

## IBM Quantum API keys

Never commit IBM Quantum API keys, account tokens, passwords, private credentials, screenshots containing tokens, `.env` files, `ibm_token.txt`, or `secrets.json`.

## Style

Keep code examples short, runnable, and teachable. Prefer clear variable names and comments that help beginners.
